# Council models

Snapshots of **council definitions** (protocol, config, agents, local skills, log layout) bundled under **`council-plugin/council-models/`**.

## Location policy

**Decision:** Keep all bundled council models here under `council-plugin/council-models/<model-name>/`. These are self-contained reference examples and are **not used by the plugin runtime**.

**Rationale:** The plugin runtime generates councils dynamically via the wizard skill. These hand-crafted examples serve as design references and should be kept as-is.

| Folder | Pattern |
|--------|---------|
| [`hub-and-spoke/`](hub-and-spoke/) | Deliberative default — Coordinator + Product Analyst + Architect + QA Strategist; voting and consensus (`decision.md` / `rejection.md` / `escalation.md`). Domain/skills may point outside this repo (see that folder's `council.config.json`). |
| [`hub-and-spoke-console/`](hub-and-spoke-console/) | Same hub-and-spoke pattern, scoped to a specific console project; `additionalDirs` is **`../../`** (project root). |
| [`adversarial-debate/`](adversarial-debate/) | Two architects, debate protocol, `recommendation.md` (bundled snapshot). |
| [`parallel-investigation/`](parallel-investigation/) | Three investigators, swarm + P2P evidence, `findings.md` (bundled snapshot). |
| [`cvi-codebase-exploration/`](cvi-codebase-exploration/) | Multi-investigator codebase analysis council. |
| [`story-refinement-pipeline/`](story-refinement-pipeline/) | **Two-stage pipeline** — refines deliberation output into Claude Code task prompts. Stage 1 (`stage-1-refinery/`): Story Analyst + Code Archaeologist + Acceptance Gate → self-contained stories. Stage 2 (`stage-2-decomposer/`): Prompt Engineer + Dependency Mapper + Test Verifier → atomic `claude --prompt-file` tasks with execution graph. |

These trees are **self-contained copies** used as reference examples; they are not generated by or consumed by the plugin.

## Config path and `additionalDirs`

- `additionalDirs` paths in each `council.config.json` are always **relative to the directory that contains** `council.config.json`.
- If you copy a model to a different location, update `additionalDirs` paths accordingly.

## Note on output paths

These example models predate the plugin's current design and use `council-log/` as their output directory. The plugin wizard generates councils that use `Sessions/` instead. This divergence is expected and does not affect runtime behavior.
